# 90 Days DevOps Learning Plan

## Current Level
Working professional with IT experience, currently transitioning into DevOps.
Basic understanding of Linux, Git, Docker, and cloud fundamentals.

## My Understanding of DevOps & Cloud
DevOps is a culture and set of practices that enable faster, reliable software delivery
through automation, collaboration, CI/CD, cloud infrastructure, and monitoring.
Cloud provides scalable infrastructure to build and deploy applications efficiently.

## Why I Am Learning DevOps
I want to move into a DevOps role where I can work on real-world infrastructure,
automation, and cloud-native systems, and grow into a strong engineer
with hands-on production experience.

## Where I Want to Reach in 90 Days
1. Deploy a production-like application on Kubernetes
2. Build CI/CD pipelines using GitHub Actions, Jenkins & Gitlab.
3. Gain confidence in Linux, Docker, AWS, and Kubernetes , AWS, Azure, Ansible and Terraform.

## Core DevOps Skills I Will Focus On
1. Linux & shell scripting
2. CI/CD pipelines and automation
3. Docker & Kubernetes fundamentals

## Time Commitment
- Weekdays: 2–2.5 hours per day
- Weekends: 4–6 hours per day

## Consistency Plan
- Daily learning with notes and hands-on practice
- Push code and progress to GitHub regularly
- Share learning updates publicly on LinkedIn to stay accountable
